# Chunk 2I Plan

## Current Resolution

Chunk 2I should follow the same quality-first workflow as the earlier Chunk 2 sub-batches, but the target mix has changed after the plant-heavy `2H` scale-up.

Confirmed on `2026-04-17`:

- Production spot-checks passed for the apex domain and sampled `2H` species pages when checked at their actual scientific-name slugs.
- `/about` also returned `200`.
- `www.isitusa.com` still failed DNS resolution in a direct check and remains a real contradiction to resolve separately.

The main planning contradiction is this:

- A mixed, category-repair shortlist does not currently reach the hoped-for `35 to 45` range without slipping back into another plant-heavy pass or lowering the source bar.
- The ready core below is intentionally smaller and cleaner than that target at `28` species, with a `10`-species backup pool.
- The backup pool exists to expand the batch only if second-source review holds up.

Current working resolution:

1. Treat the ready core as the actual `2I` drafting pool.
2. Use the backup list only after second-source review.
3. Do not pad the batch with thin wildlife or disease profiles just to hit a round number.

## Drafting Order

If drafting starts immediately after this plan, use this order:

1. Insects with the clearest extension guidance.
2. Wildlife with the clearest report or do-not-move framing.
3. Fungi-disease profiles that already have clean official symptom language.
4. The smaller plant tail only after the non-plant core is secure.

## Guardrails For Drafting

- Keep the stronger second-source rule for `Mediterranean Gecko`, `Brown Trout`, and `Chocolate Vine`.
- Do not let fish and reptile profiles drift into generic aquarium-release boilerplate.
- Keep disease action language conservative and practical, especially where homeowner treatment is not appropriate.
- Do not expand the batch with backup species until the ready core still reads distinct after the first drafting pass.
